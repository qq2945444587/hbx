版本更新提醒

由于系统受到了广泛的关注和推荐，默认赠与的请求调用额度已经无法满足广大用户的需求。经过再三考虑，我们上线了商城功能，您可以在商城购买调用额度。

借此，为了您的利益考虑，我们增加接口的加密，保证用户的利益不受损害，具体变化如下：

  • 增加了 app_secret，您可以存储起来，在接口调用中加密使用
  • Header中增加了sign 和 timestamp 两个内容
timestamp 为时间戳，是从1970-01-01日到现在的秒数
sign为加密的字符串，计算方法为：md5(appKey + appSecret + modelVersion + prompt + timestamp)

详细的内容见下面的 QuickStart -> 请求 部分

简介

我们的API几乎可以应用于任何涉及理解或生成自然语言或代码的任务。我们提供了一系列具有不同参数级别的模型，适用于不同的任务，以及微调您自己的自定义模型的能力。这些模型可用于从内容生成到语义搜索和分类的所有内容。

关键概念

提示和补全

补完是我们API的核心。它为我们的模型提供了一个简单的文本输入、文本输出界面，非常灵活和强大。您输入一些文本作为提示，模型将生成一个文本补全，用于尝试匹配您提供的任何上下文或模式。 例如，如果您给出了提示的，床前明月光，下文很大概率就会接 疑是地上霜。

设计你的提示本质上是你如何“编程”模型，通常是通过提供一些说明或几个例子。这与大多数其他为单一任务设计的 NLP 服务不同，例如情感分类或命名实体识别等。相反，补全功能几乎可用于任何任务，包括内容或代码生成、总结、扩展、对话、创意写作、风格转换等。

Tokens

我们的模型通过将文本分解为token来理解和处理文本。token可以是词或只是字。例如，“北京你好” 被分解为token “你”、“好” 、“北” 、“京”，而像“朋友”这样的短而常见的词是单个token。

在给定 API 请求中处理的token数量取决于您的输入和输出的长度。根据粗略的经验，1个token 1.42个字。要注意的一个限制是，您的文本提示和生成的补全组合不得超过模型的最大上下文长度2048 个。

模型

该API由一系列具有不同功能和价格点的模型提供支持。我们的基础GPT-3模型被称为天枢，增强版称为瑶光。要了解更多信息，请访问我们的模型引擎相关内容。

QuickStart

请求

您可以将下面的命令粘贴到终端中以运行您的第一个API请求。请务必使用秘密API密钥替换YOUR_API_KEY,TIMESTAMP,SIGN。

curl https://openapi.singularity-ai.com/api/v2/generateByKey \
-H "Content-Type: application/json" \
-H "timestamp: TIMESTAMP" \
-H "sign: SIGN" \
-d '{
    "data": {
        "prompt": "从前有座山",
        "model_version": "benetnasch_common_gpt3",
        "param": {
                   "top_p": 1000,
                   "top_k": 1000,
                   "repetition_penalty":  1.5,
                   "length_penalty":  5,
                   "min_len":  [ "一二三", "一二三" ],
                   "bad_words": [ ["EOS"] ],
                   "end_words":  1.0,
                   "my":  false,
              }
         }
  }'

这个请求查询瑶光引擎，以提示“从前有座山”开始完成文本。max_tokens参数设置API将返回多少个token的上限。你应该得到如下的响应:

{
        "code": "200",
        "codeMsg": "",
        "param": {
                   "reply": ，山上有个庙。 老和尚和小沙弥一起住持着这座寺院。,
                   "status": 3,
              }
  }

现在你已经生成了你的第一个任务。
